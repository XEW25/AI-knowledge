### Abstract / core thesis
The paper argues that long-horizon agents degrade as interaction history grows because of context-window limits, token cost, and long-context failures. Existing memory methods often treat memory as an external retrieval system, making memory use passive and not jointly optimized with the agent objective.

The paper’s core move is to make memory management an intrinsic agent capability. The agent explicitly emits memory summaries during interaction through a `<mem>` action, alongside `<think>` and `<tool_call>`. These memory actions are then trained with Self-Memory Policy Optimization (MemPO), which provides additional reinforcement-learning credit specifically to memory tokens. Memory quality is rewarded by how much it increases the model’s conditional probability of generating the correct answer.

### Main claims
- Memory should be internalized into the policy, not treated only as an external retrieval component.
- Long-horizon agents benefit when memory writing becomes an explicit action.
- Memory tokens need denser credit assignment than a sparse final-answer reward can provide.
- A memory-aware RL objective improves long-horizon performance while reducing token usage.

### Method sketch
The interaction trajectory contains:
- `<mem>` for memory summary
- `<think>` for reasoning
- `<tool_call>` for tool use
- `<information>` for tool output
- `<answer>` for the final answer

The method combines:
- a trajectory-level binary reward for final correctness and output format
- a memory-level reward based on whether a memory summary raises the conditional probability of the correct answer

For memory tokens, the advantage is the sum of trajectory-level and memory-level advantages. Other tokens only receive the trajectory-level advantage.

### Experiments and headline results
The paper evaluates on multi-objective long-horizon tasks under local wiki search and online web search, plus appendix evaluations on single-objective QA and deep-research-style benchmarks.

Reported headline numbers:
- +25.98 absolute F1 over the base model
- +7.1 absolute F1 over the previous SOTA baseline
- 67.58% / 73.12% token reduction

### Why this paper matters
This paper is important because it treats memory not merely as retrieval infrastructure but as an explicit, trainable part of agent policy. It reframes memory writing as something the agent should learn to do well, rather than something a human-engineered memory system does on the agent’s behalf.

### Limitations noted in the paper
The paper notes that group-based memory-advantage estimation may be biased because step-aligned memory states across rollouts are not perfectly comparable. It also relies on gold answers during training to compute the memory usefulness signal.
