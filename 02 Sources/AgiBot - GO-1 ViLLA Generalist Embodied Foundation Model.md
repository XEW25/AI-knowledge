> **定位**：GO-1 提出 **ViLLA（Vision-Language-Latent-Action）** 框架——三段式 hierarchy。系统级接口是**离散 latent action token（VQ-VAE）**；VLM↔Latent Planner 是**范式 A 式 joint attention（逐层条件化）**。核心创新：latent action token 让模型能从无标注 web 视频学习动作表征。

## 架构精度：三段式 + Latent Action Token 接口（已核实论文）

数据流：**VLM → Latent Planner → Action Expert**

### Latent Action Model (LAM) —— "出题人"
- 结构：逆动力学编码器 `I(z_t | I_t, I_{t+H})` + 正动力学解码器 `F(I_{t+H} | I_t, z_t)`
- 从连续帧对（当前帧 + H 步后帧）提取 latent action，**VQ-VAE 量化成离散 token**（k=4 token/动作）
- **关键**：只需**帧对**即可学，无需机器人动作标签 → 能从**无标注 web 视频 + 人类视频**学"帧间变化的动作词汇表"（codebook）
- 角色：训练时充当"出题人"，用未来帧给出标准答案 token

### Latent Planner（核心）—— "答题人"

**角色**：在压缩的 latent action 空间做**时序推理**，搭桥 VLM 图文理解 →（latent action token）→ action expert 低层控制。
> 原文："The latent planner conducts temporal reasoning through latent action prediction, bridging the gap between image-text inputs and robot actions generated by the action expert."

**输入**：
- 多视角图像 `(I_t^h, I_t^l, I_t^r)`（头/左/右）+ 语言指令 `l`
- VLM backbone（InternVL2.5-2B）的**逐层特征**：24 层 transformer，"layer-by-layer conditioning from the VLM backbone with full bidirectional attention"（→ **范式 A** 式逐层 joint）
- 二手补充：planner 与 VLM 共享架构但用独立 FFN + Q/K/V/O（MoE 式独立权重）

**输出**：
- **k=4 个离散 latent action token** `z_t = [z_t^0, …, z_t^{k-1}]`，即 `P(z_t | I_t^h, I_t^l, I_t^r, l)`

**训练 vs 推理（精髓）**：
```
训练：目标 token 由 LAM 逆动力学编码器从「当前+未来帧」算出
      z_t := I(I_t^h, I_{t+H}^h)   ← 偷看未来；作为 pseudo-label 监督 planner
      planner 学着「只看当前」逼近它（cross-entropy）
推理：未来帧不可得 → planner 只凭「当前观测+指令」预测 z_t
      ← 这才是真正的"规划"（预测该发生什么，而非看到后再编码）
```

### Action Expert —— "解码器"
- 原文："shares the same architectural framework as the latent planner"
- diffusion objective 建模连续动作；**条件化于 latent action token** 做去噪 → 低层控制信号
- 分工：planner 管"该发生什么"（embodiment 无关计划），action expert 管"怎么用这个身体 actuate"（embodiment 相关）→ 利于跨本体泛化

### 完整数据流
```
推理：当前多视角图像 + 指令
        ↓ VLM (InternVL2.5-2B) 逐层特征
   Latent Planner (24层) ──→ 预测 k=4 个 latent action token z_t
        ↓ z_t 作为条件
   Action Expert (diffusion) ──→ 去噪生成连续低层动作
```

**两层耦合定位**：
- **系统级接口** = latent action token（离散，VQ-VAE）——介于"单 latent 向量"（Helix）和"语言子任务"（G0）之间
- **VLM↔Latent Planner 内部耦合** = 范式 A（joint，逐层）

详见 [[Embodied Brain Models]] 两层耦合框架。

## Why It Matters

- **latent action token 是从视频学习的关键**：用 VQ-VAE 学"动作词汇表"，绕开真机动作标签稀缺问题
- **AgiBot World 数据集**：100 万+ 轨迹开源，是中国具身数据规模派的代表
- **部署导向路线 2 的代表**：显式多段式 + 压缩 token 接口
